1. Achados Críticos

Crítico Coluna pdf_url referenciada no código, inexistente no DB

O service layer magazineService.ts define MagazineRow.pdf_url e a lê/escreve em 4 locais (linhas 36, 84, 336, 534). Essa coluna não existe na tabela magazines.

Impacto: Qualquer operação de update que inclua pdfUrl no patch lança erro 400 do PostgREST em runtime. Leituras retornam null silenciosamente (PostgREST ignora colunas extras no SELECT).

src/services/magazineService.ts:36,84,336,534

src/services/__tests__/magazinePublish.fuzz.test.ts:59

Crítico Conector MCP Self-Hosted aponta para DB errado

O conector mcp__SUPABASE_SELF_HOSTED_-_MCP retorna 0 linhas para todas as tabelas magazine_* e erro “relation does not exist” no describe_table. Ele não está conectado ao projeto Gold doufsxqlfjyuvxuezpln.

Impacto: Qualquer automação ou auditoria via MCP self-hosted opera sobre um banco diferente, gerando falsos negativos.

Crítico Tipo branding.primaryColor vs colors.primary

O guard magazine-guard.ts (linhas 43-50) valida branding.primaryColor, mas o default da coluna magazines.branding no DB usa a estrutura colors.primary:

jsonb_build_object('colors', jsonb_build_object('primary', '#2e4a3a', ...))

O tipo MagazineClientBranding em src/types/magazine.ts também usa colors.primary. O guard nunca encontra primaryColor e pula a validação.

src/lib/security/magazine-guard.ts:43-50
