## Bugs Detalhados

### 🔴 BUG-A01 — P0 CRÍTICO: Email transacional nunca enviado

**Arquivo:** `supabase/functions/send-transactional-email/index.ts`
**Impacto:** 100% dos emails (quote_sent, quote_approved, order_created) são silenciosamente descartados
**Root cause:** A função gera o HTML do email, loga em `workspace_notifications` e retorna `{ success: true, message: "Email queued" }` — porém não há nenhum provedor de envio integrado (Resend, SendGrid, Postmark, etc.). O comentário no código diz _"actual sending happens when email domain is configured"_ mas esse código nunca foi escrito.
**Fix aplicado:** Integração com Resend via `resolveCredential('RESEND_API_KEY')`. Se a chave não estiver configurada, o comportamento anterior (log + pseudo-sucesso) é mantido como fallback gracioso com warning no log.
**Ação pós-deploy:** Configurar `RESEND_API_KEY` em `/admin/conexoes`.

---

### 🔴 BUG-A02 — P1 ALTO: SSRF em `analyze-logo-colors`

**Arquivo:** `supabase/functions/analyze-logo-colors/index.ts`
**Impacto:** Qualquer usuário autenticado pode forçar o servidor a fazer fetch em qualquer URL (incluindo `http://169.254.169.254/latest/meta-data/` em ambientes cloud)
**Root cause:**
```typescript
if (imageBase64.startsWith('http://') || imageBase64.startsWith('https://')) {
  const imgResponse = await fetch(imageBase64); // ← SSRF sem allowlist
```
**Fix aplicado:** Allowlist de domínios permitidos para URLs. URLs que não estiverem na lista retornam 400.
**Domínios permitidos:** `supabase.co`, `cloudflare.com`, `imagedelivery.net`, `promobrindes.com.br`, `promogifts.com.br`, `amazonaws.com`, `spotgifts.com.br`, `asiaimport.com.br`, `somarcas.com.br`.

---

### 🔴 BUG-A03 — P1 ALTO: `LOVABLE_API_KEY` bypassa credential SSOT

**Arquivo:** `supabase/functions/analyze-logo-colors/index.ts`
**Impacto:** Rotações de chave via `/admin/conexoes` não têm efeito. Isolate precisa ser reiniciado.
**Root cause:** `Deno.env.get('LOVABLE_API_KEY')` em vez de `resolveCredential('LOVABLE_API_KEY')`.
**Fix aplicado:** Substituição por `resolveCredential()` com padrão SSOT (DB-first → env fallback).

---

### 🔴 BUG-A04 — P1 ALTO: `image-proxy` bloqueia fornecedores XBZ, Asia, Cloudflare

**Arquivo:** `supabase/functions/image-proxy/index.ts`
**Impacto:** Imagens de qualquer fornecedor que não seja `spotgifts.com.br` são bloqueadas com 403
**Root cause:**
```typescript
const ALLOWED_DOMAINS = [
  'www.spotgifts.com.br',
  'spotgifts.com.br',
  // XBZ, Asia Import, Só Marcas, Cloudflare Images ausentes
];
```
**Fix aplicado:** Adicionados os domínios de todos os fornecedores ativos.

---

### 🔴 BUG-A05 — P1 ALTO: `promo-gifts-beta.vercel.app` fora do allowlist de referer

**Arquivo:** `supabase/functions/image-proxy/index.ts`
**Impacto:** Ambiente de staging/desenvolvimento rejeita todas as imagens com 403 (hotlink protection)
**Root cause:** `ALLOWED_REFERER_HOSTS` não inclui `promo-gifts-beta.vercel.app`.
**Fix aplicado:** Adicionado o hostname de staging e `vercel.app` genérico para previews.

---

### 🟡 BUG-A06 — P1 ALTO: Corpo do erro da CNPJA API descartado

**Arquivo:** `supabase/functions/cnpj-lookup/index.ts`
**Impacto:** Erros como CNPJ inativo (422), rate limit (429) ou não encontrado (404) retornam apenas o código HTTP sem detalhe
**Root cause:**
```typescript
if (!response.ok) {
  await response.text(); // resultado nunca usado
  console.error(`CNPJÁ API error [${response.status}]`);
  // mensagem ao usuário: genérica
}
```
**Fix aplicado:** Body do erro é lido, parseado (se JSON) e incluído na resposta ao usuário e no log.

---

### 🟡 BUG-A07 — P2 MÉDIO: Bypass interno usa substring match em `webhook-inbound`

**Arquivo:** `supabase/functions/webhook-inbound/index.ts`
**Impacto:** Potencial bypass de rate limit se uma string que contenha o service key for injetada no Authorization header
**Root cause:**
```typescript
const isServiceRole = authHeader.includes(serviceKey) || ...
//                             ^^^^^^^^ substring match
```
**Fix aplicado:** Comparação exata após strip do prefixo `Bearer `.

## Bugs Documentados (fix pendente)

### BUG-A08 — P1 ALTO: Token Dropbox expira sem refresh
**Arquivo:** `supabase/functions/dropbox-list/index.ts`
**Detalhe:** Dropbox tokens OAuth2 expiram em ~4h. A função usa token estático de longa duração sem mecanismo de refresh. Quando expirar, toda integração Dropbox falha silenciosamente com 401 (retornado como 500 para o usuário).
**Mitigação:** Usar Dropbox App Access Token (não expira) ou implementar refresh_token flow via edge function dedicada.

---

### BUG-A09 — P1 ALTO: `SELLER_EMAIL_MAP` hardcoded em `sync-quote-bitrix`
**Arquivo:** `supabase/functions/sync-quote-bitrix/index.ts`
**Detalhe:** Mapa de email → ID Bitrix hardcoded no código. Novo vendedor = PR + deploy. Vendedor demitido = risco de ID órfão.
**Mitigação sugerida:** Mover o mapa para a tabela `integration_credentials` (como JSON) ou para `profiles` (coluna `bitrix_user_id`).

---

### BUG-A10 — P2 MÉDIO: `WEBHOOK_INBOUND_V1_COMPAT_ENABLED` default false bloqueia n8n
**Arquivo:** `supabase/functions/webhook-inbound/index.ts`
**Detalhe:** Webhooks v1 (formato flat, sem envelope `{ event, data, occurred_at }`) são bloqueados por padrão com 426. O n8n envia webhooks v1 por padrão. Para liberar: setar `WEBHOOK_INBOUND_V1_COMPAT_ENABLED=true` E adicionar o issuer ao `WEBHOOK_INBOUND_V1_ALLOWLIST`.
**Ação:** Verificar se algum workflow n8n existente usa `webhook-inbound`. Se sim, ativar v1 compat imediatamente.

---

### BUG-A11 — P2 MÉDIO: `logDecision` aguardado no path de sucesso do AI Router
**Arquivo:** `supabase/functions/_shared/ai-router/index.ts`
**Detalhe:** `await logDecision(...)` adiciona latência de DB a toda chamada AI bem-sucedida. `logAiUsage` já é fire-and-forget corretamente. `logDecision` deveria ser também.
**Fix simples:** `logDecision(...).catch(e => console.error(...))` no path de sucesso.

---

### BUG-A12 — P2 MÉDIO: Três versões de Zod nas edge functions
**Impacto:** Comportamentos de validação podem divergir entre functions
```
elevenlabs-tts: https://deno.land/x/zod@v3.22.4/mod.ts
cnpj-lookup:    npm:zod@3.23.8
dropbox-list:   https://esm.sh/zod@3.23.8
```
**Fix:** Padronizar para `npm:zod@3.23.8` em todas as functions (ou usar `_shared/zod-validate.ts` que já existe).

---

### BUG-A13 — P2 MÉDIO: `OffscreenCanvas` pode falhar em Edge runtime
**Arquivo:** `supabase/functions/generate-mockup/index.ts`
**Detalhe:** `OffscreenCanvas` requer flag `--unstable` em algumas versões do Deno e pode não estar disponível no runtime Supabase Edge. Sem fallback, retorna 500 genérico.
**Mitigação:** Adicionar try/catch específico para `OffscreenCanvas` não disponível, com mensagem clara.

---

### BUG-A14 — P2 MÉDIO: `Retry-After` hardcoded em `external-fetch.ts`
**Arquivo:** `supabase/functions/_shared/external-fetch.ts`
**Detalhe:** Todo `circuitOpenResponse` retorna `Retry-After: 60`. CNPJA tem rate limit de 35 req/dia (pode precisar de horas), Dropbox tem recovery em segundos.
**Fix:** Aceitar `retryAfterSeconds` como parâmetro opcional com default 60.

---

### BUG-A15 — P3 BAIXO: `var` em `compositeImages()` de `generate-mockup`
**Fix trivial:** Substituir `var pa`, `var sx`, etc. por `const`/`let`.

### BUG-A16 — P3 BAIXO: Mock CNPJ sem guard de produção
**Arquivo:** `supabase/functions/cnpj-lookup/index.ts`
O CNPJ `00000000000191` (Banco Central do Brasil) retornava dados mock hardcoded sem verificar se está em ambiente de testes. **Fix incluído neste PR:** guard via variável `ENVIRONMENT=production`.

### BUG-A17 — P3 BAIXO: Dual Supabase client em `send-transactional-email`
A função cria dois clientes: um com `ANON_KEY + auth header` para validar o usuário, e outro com `SERVICE_ROLE_KEY` para o insert. Pode ser consolidado em um único service client com verificação de JWT manual.

### BUG-A18 — P3 BAIXO: AI Router sem hint de admin quando routing ausente
Quando `getRouting(functionName)` retorna null, o erro é `"No active routing for function X"` sem sugestão de onde configurar. Melhorar com: `Configure em /admin/conexoes > AI Models > Routing`.

### BUG-A19 — P3 BAIXO: Paginação Dropbox ignorada
**Arquivo:** `supabase/functions/dropbox-list/index.ts`
A API Dropbox retorna `has_more: true` e `cursor` para pastas com muitos arquivos. A função retorna esses campos mas nunca os usa para buscar páginas subsequentes. Pastas com >2000 itens são silenciosamente truncadas.

### BUG-A20 — P3 BAIXO: `send-transactional-email` sem validação de domínio do destinatário
Não há validação de que o `recipient_email` pertence ao domínio da empresa. Qualquer usuário autenticado pode triggerar envio de email para qualquer endereço externo.
