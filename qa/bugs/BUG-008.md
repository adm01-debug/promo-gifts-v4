ID: BUG-008 | Area: Database/Types | Severity: P1
Route/File: 30+ hooks and services across the codebase
Steps to reproduce: Run tsc --noEmit
Result: ~150 TypeScript errors — supabase.from('table') resolves to `never` for tables not in generated types
Expected: Type-safe access to all tables
Root cause: Supabase generated types (types.ts) out of sync with actual database schema. Many tables exist in DB but not in generated types.
Fix: Migrated all affected `supabase.from()` calls to `untypedFrom()` from `src/lib/supabase-untyped.ts`
Test: TypeScript compiler
