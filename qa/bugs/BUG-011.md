ID: BUG-011 | Area: Security | Severity: P2
Route/File: Supabase database — public schema functions
Steps to reproduce: Query information_schema.routines for SECURITY DEFINER functions
Result: 30+ functions with SECURITY DEFINER (bypasses RLS)
Expected: Functions should use SECURITY INVOKER unless explicitly needed
Root cause: Many functions created with DEFINER to perform admin operations (triggers, enrichment, archival)
Fix: Audit each DEFINER function to verify it validates auth.uid() internally or is only called from trusted contexts (triggers, edge functions with service_role key)
Test: Manual audit + RLS integration tests
Note: Not all DEFINER functions are vulnerabilities — triggers and admin functions legitimately need DEFINER. But each should be verified.
