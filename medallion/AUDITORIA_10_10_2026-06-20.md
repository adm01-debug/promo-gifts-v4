## 4. Migration pronta — NÃO executada (aguarda conferência fiscal)

Após você/contador confirmar as alíquotas, aplico isto via `sim → apply → validate`
(fill-only, flags de pipeline). Preenche `ncm_codes` e propaga Silver→Gold:

```sql
-- 1) Alíquotas confirmadas na tabela de referência (exemplo: 9 NCMs sourced)
UPDATE ncm_codes SET ipi_rate = v.rate,
       source = 'tipi_fazcomex_2025', notes = 'TIPI conferido 2026-06-20', updated_at = now()
FROM (VALUES
  ('48202000', 0.00), ('96080010', 13.00), ('75089090', 0.00),
  ('48192000', 9.75), ('63051000', 9.75), ('42023200', 6.50),
  ('48201000', 9.75), ('94052100', 9.75), ('95049090', 13.00)
) AS v(code, rate)
WHERE regexp_replace(ncm_codes.code,'\D','','g') = v.code
  AND ncm_codes.ipi_rate IS NULL;

-- 2) Propagação fill-only (mesma lógica autoritativa da etapa 2)
SELECT set_config('app.write_source','pipeline', true);
SELECT set_config('app.bulk_import_mode','true', true);
UPDATE produtos_padronizacao pp SET ipi_rate = c.ipi_rate, updated_at = now()
  FROM ncm_codes c
  WHERE pp.ipi_rate IS NULL AND c.ipi_rate IS NOT NULL
    AND regexp_replace(pp.ncm_code,'\D','','g') = regexp_replace(c.code,'\D','','g');
UPDATE products p SET ipi_rate = c.ipi_rate
  FROM ncm_codes c
  WHERE p.ipi_rate IS NULL AND c.ipi_rate IS NOT NULL
    AND regexp_replace(p.ncm_code,'\D','','g') = regexp_replace(c.code,'\D','','g');
```
